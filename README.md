# 🪸 Rakshak — Personal Dependency Health & Security Agent

Rakshak is a personal dependency agent built for the WeMakeDevs hackathon. It monitors your code repositories, extracts dependency version pins, queries the **Coral query engine** under WSL, calculates risk scores, retrieves CVE/GHSA advisories, searches Hacker News & Dev.to articles, and provides a beautiful interactive web dashboard along with a conversational Telegram bot.

---

## 🚀 Key Features
1. **Repository Dependency Discovery:** Periodically scans your GitHub repositories to discover `package.json` (NPM), `requirements.txt` (PyPI), and `Cargo.toml` (Cargo) manifests.
2. **Ecosystem Registry Queries:** Integrates with Coral under WSL to query package registry metadata (`deps_dev` and `crates_io` tables).
3. **Real-time Vulnerability Feeds:** Performs automated security checks against Google's `osv` tables matching your exact versions.
4. **Community Intelligence:** Searches Hacker News discussions to identify early warnings or breaking changes, and queries Dev.to to find migration guides.
5. **Interactive Risk Heatmap:** Shows a grid of all project dependencies colored by risk (Critical, Warning, Healthy), with detail drawers.
6. **Conversational AI Q&A:** A plain-text search bar that lets you ask questions like "which packages have critical vulnerabilities?" and retrieves answers generated by **Gemini 2.5 Flash** directly from query outputs.
7. **Bidirectional Telegram Bot:** Receive critical alerts instantly via Telegram, broadcast migration guides to chat, or run slash commands (`/health`, `/security`, `/status`, `/refresh`, `/fix`) to manage your dependencies.

---

## 🛠️ Prerequisites
- **Python 3.11+** installed on Windows.
- **Node.js 18+** installed on Windows.
- **WSL (Windows Subsystem for Linux)** with **Coral** installed and reachable.
- **Coral Sources:** Ensure the following community sources are added to Coral:
  - `deps_dev` (manifest at `sources/community/deps_dev/manifest.yaml`)
  - `osv` (manifest at `sources/community/osv/manifest.yaml`)
  - `crates_io` (manifest at `sources/community/crates_io/manifest.yaml`)
  - `hn` (manifest at `sources/community/hn/manifest.yaml`)
  - `devto` (manifest at `sources/community/devto/manifest.yaml`)

## ⚡ Quickstart Guide

### 1. Install Python Dependencies
```bash
pip install -r requirements.txt
```

### 2. Configure Environment Variables
Copy `.env.example` to `.env` and fill in the values (leave keys empty if you want to skip integrations or run in demo mode):
```bash
copy .env.example .env
```

### 3. Initialize & Populate SQLite Database
Run the setup command to initialize SQLite database tables:
```bash
python -c "import sys; sys.path.insert(0, './'); from backend.database import Base, engine; Base.metadata.create_all(engine); print('DB Initialized')"
```

To pre-load realistic mock data for demo fallback mode:
```bash
# Run backend server first, then hit reset
curl -X DELETE http://localhost:8000/api/demo/reset
```

### 4. Run the Servers

#### Start Python Backend
```bash
# From the rakshak directory
uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
```
This runs the FastAPI API server, background scheduler, and Telegram bot concurrently!

#### Start React Frontend
```bash
# Open a new terminal inside rakshak/frontend
npm install
npm run dev
```
Open [http://localhost:5173](http://localhost:5173) in your browser to view the dashboard!

## 💡 Demo Safety Nets & Offline Mode
If you do not have Coral or WSL set up, or are offline, set `RAKSHAK_DEMO_FALLBACK=1` inside the `.env` file. Rakshak will automatically intercept all Coral SQL queries and return pre-populated high-fidelity data from `demo/seed.sql` for a flawless demonstration.
